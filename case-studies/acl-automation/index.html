Automated SKU validation and matrix reconciliation using Google Apps Script — replacing manual row-by-row review with enforced business rules, reducing data entry errors and ensuring consistency across the accessory control list.

Core Capability

Validation Rules Enforced

🔑
SKU Uniqueness
Prevents duplicate SKU entries across the matrix. Flags any submission that would create a conflict.
📋
Master List Cross-Check
Every submitted SKU is validated against the master reference list. Unrecognized SKUs are rejected before they enter the system.
🏷️
Category Consistency
Ensures OEM, product category, and revision fields are consistent with the master record for that SKU.
⚠️
Revision Conflict Detection
Detects when a new submission conflicts with an existing revision entry — prompts review instead of silent overwrite.
📅
Required Fields Check
Validates that all mandatory fields are populated before writing to the sheet. Incomplete submissions are blocked at input.
🔄
Auto-Sync to Master
Validated entries are automatically appended to the master list — no manual copy-paste between sheets.
How It Works

Submission Flow

1
Input via Google Form / Sheet
Team members submit new ACL entries through a structured Google Form or directly into the input sheet. No free-form text — dropdowns and constrained inputs reduce typos at the source.
2
onEdit Trigger Fires
A Google Apps Script onEdit trigger fires on each new row submission, passing the entry through the validation chain automatically — no manual step required.
3
Rule Chain Validation
Entry is checked sequentially: required fields → SKU exists in master → no duplicate → category matches → revision check. First failure stops the chain and flags the row.
4
Pass / Fail Routing
Validated entries are written to the master ACL sheet. Failed entries are highlighted in red with an inline note explaining which rule was violated.
5
Notification (Optional)
On validation failure, an email summary is sent to the submitter with the specific rule that failed — enabling quick self-correction.
Code Sample

Validation Chain (Apps Script)

Google Apps Script · ACL Validation 
function validateACLEntry(row) {
  const master = getMasterList();
  const sku    = row["sku"];

  // 1. Required fields
  const required = ["sku", "category", "oem", "revision"];
  for (const field of required) {
    if (!row[field]) return fail(`Missing required field: ${field}`);
  }

  // 2. SKU must exist in master list
  if (!master.has(sku)) {
    return fail(`SKU not found in master list: ${sku}`);
  }

  // 3. No duplicates
  if (isDuplicate(sku)) {
    return fail(`Duplicate SKU entry detected: ${sku}`);
  }

  // 4. Category must match master record
  const ref = master.get(sku);
  if (ref.category !== row["category"]) {
    return fail(`Category mismatch for ${sku}`);
  }

  // All checks passed — write to master ACL
  appendToMaster(row);
  return { ok: true };
}
